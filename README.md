# 📱 Unity Android UI 관리 프로젝트

- **Unity Version**: 2022.3.20f1  
- **Build Target**: Android  
- 🎥 [시연 영상 보기 (YouTube)](https://www.youtube.com/watch?v=12Lw8zUfoOY)

---

## 🧩 핵심 기술

### 🔄 UI 다중 오브젝트 풀링
- 생성된 UI 오브젝트를 활성화/비활성화로 관리하여 빠른 UI 전환
- 가비지 컬렉션 발생 최소화 → **메모리 단편화 방지**
- 모든 UI 창을 프리팹화하여 하이어라키에서 분리, **모듈화 및 협업 최적화**
- **Dictionary 기반 오브젝트 풀링**: 탐색 효율성 확보 (List, Queue, Stack보다 유리)

### 🎯 디자인 패턴 적용
- **MVP 패턴 적용**: 각 UI 창의 모델/뷰 의존성 제거 → 코드 유지보수성 향상
- **싱글톤 패턴**으로 공통 리소스 관리

### 💾 데이터 저장 및 구조화
- **JSON 기반 유저 데이터 관리**
- 캐릭터와 무기 간의 순환 참조 방지를 위해 이름을 **고유 키**로 사용
- Android RAM 환경을 고려하여, **전체 데이터 대신 유저 데이터만 메모리에 로드**
- 캐릭터 및 장비 옵션 변경 시 자동 저장 기능 구현

## ⚙️ 주요 기능

### 🧙‍♂️ 캐릭터 및 장비 시스템
- 캐릭터, 무기, 방어구, 스킬 등 클래스 구조화 및 `get/set` 프로퍼티 기반 접근
- 보유 캐릭터와 기본 정보를 슬롯 형식으로 UI에 출력
- 캐릭터 상세 스탯 및 장비 상세 정보 뷰 제공

### 🧭 뷰 전환
- 기사단 뷰(KnightsView), 캐릭터 정보 뷰(CharacterInfoView), 장비 창 뷰(EquipmentView) 간 이동 구현

### 🔍 캐릭터 정렬 및 필터링
- 레벨 및 등급 기준 내림차순 정렬
- 속성과 직업 기준 필터링
- 정렬/필터링 조합 로직:


### 🗡️ 무기 장착 및 관리
- 무기 장착/해제 기능 및 직업 제한 표시
- 무기 레벨업, 연마, 분해 기능
- 장착 시 슬롯 자동 갱신 및 중복 착용 방지

## 🛡️ 기타 구현 사항

- **DataModel의 Save() 메서드**는 외부에서 직접 호출 불가하도록, **Delegate와 Event를 통해 간접 접근** 처리
- **순환 참조 방지**: `Character`와 `Weapon` 객체는 JSON 변환 시 이름을 고유 식별자로 사용하여 참조 관계 해결
